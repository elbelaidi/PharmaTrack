Pharmacy Management System API Documentation

This document provides information about the RESTful API endpoints available in the Pharmacy Management System.

Authentication

The API currently uses the same authentication mechanism as the web application. You need to be logged in to access the API endpoints.

API Endpoints

Users

GET /api/users

Retrieves a list of all users.

Response

[
  {
    "id": 1,
    "username": "admin",
    "fullName": "Admin User",
    "email": "admin@example.com",
    "phoneNumber": "1234567890",
    "role": "ADMIN",
    "active": true,
    "createdAt": "2023-01-01 00:00:00",
    "lastLogin": "2023-01-01 00:00:00"
  },
  ...
]
GET /api/users/{id}

Retrieves a specific user by ID.

Response

{
  "id": 1,
  "username": "admin",
  "fullName": "Admin User",
  "email": "admin@example.com",
  "phoneNumber": "1234567890",
  "role": "ADMIN",
  "active": true,
  "createdAt": "2023-01-01 00:00:00",
  "lastLogin": "2023-01-01 00:00:00"
}
POST /api/users

Creates a new user.

Request Body

{
  "username": "newuser",
  "password": "password123",
  "fullName": "New User",
  "email": "newuser@example.com",
  "phoneNumber": "1234567890",
  "role": "PHARMACIST",
  "active": true
}

Response

{
  "id": 3,
  "username": "newuser",
  "message": "User created successfully"
}
PUT /api/users/{id}

Updates an existing user.

Request Body

{
  "username": "updateduser",
  "password": "newpassword",
  "fullName": "Updated User",
  "email": "updated@example.com",
  "phoneNumber": "9876543210",
  "role": "PHARMACIST",
  "active": true
}

Response

{
  "message": "User updated successfully"
}
DELETE /api/users/{id}

Deletes a user.

Response

{
  "message": "User deleted successfully"
}

Error Handling

All API endpoints return appropriate HTTP status codes:

  • 200 OK: The request was successful
  • 201 Created: A new resource was created successfully
  • 400 Bad Request: The request was invalid or cannot be served
  • 404 Not Found: The requested resource does not exist
  • 500 Internal Server Error: An error occurred on the server

Error responses include a JSON object with an error message:

{
  "error": "Invalid user ID"
}

Notes

This API is designed to be used with the Pharmacy Management System web application. It provides programmatic access to the same data that is available through the web interface.
